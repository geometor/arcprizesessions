Observations:

Both examples demonstrate a consistent pattern:

  • The input grids contain one or more L-shaped objects formed by azure (8) pixels. The rest of the grid is white (0).

  • The output grids are the same size as the input grids. They contain the original azure L shapes plus additional blue (1) pixels.

  • The transformation involves adding a single blue pixel adjacent to each L shape. Specifically, a blue pixel is added to the right of the topmost azure pixel(s) in the L and to the left of the bottommost azure pixel(s) in the L.

Proposed Transformation Story:

  1. Identify L-shapes: Locate all distinct L-shaped objects composed of azure (8) pixels within the input grid.

  2. Add Blue Pixels: For each azure L-shape, add a single blue (1) pixel to the immediate right of the topmost azure pixel(s) in the L and another blue pixel to the immediate left of the bottommost azure pixel(s) in the L.

Specifically I need to make the code aware that an L can be in these orientations:

  • 0 degrees (upright L)

  • 270 degrees clockwise The code will first detect all azure pixels. Then iterate through them and examine adjacent neighbors to determine if they are part of an L. The L is then classified by orientation, and the correct blue pixels are added. The output should be a modified version of the original grid with the added blue pixels.

import numpy as np

def find_l_shapes(grid):
    azure_pixels = np.argwhere(grid == 8)
    l_shapes = []

    for pixel in azure_pixels:
        x, y = pixel
        neighbors = []
        for dx in [-1, 0, 1]:
            for dy in [-1, 0, 1]:
                if (dx, dy) == (0, 0):
                    continue
                nx, ny = x + dx, y + dy
                if 0 <= nx < grid.shape[0] and 0 <= ny < grid.shape[1] and grid[nx, ny] == 8:
                    neighbors.append((nx, ny))


        if len(neighbors) >= 1:
            #check if part of existing L
            is_new = True
            for l in l_shapes:
                if pixel.tolist() in l:
                    is_new = False
            if is_new:
                current_l = [pixel.tolist()]
                for n in neighbors:
                    current_l.append(n.tolist())
                    n_x, n_y = n
                    for dx2 in [-1, 0, 1]:
                        for dy2 in [-1, 0, 1]:
                            if (dx2, dy2) == (0, 0) or (dx2,dy2) == (-dx, -dy):
                                continue
                            nx2, ny2 = n_x + dx2, n_y + dy2
                            if 0 <= nx2 < grid.shape[0] and 0 <= ny2 < grid.shape[1] and grid[nx2, ny2] == 8 and [nx2, ny2] not in current_l:
                                current_l.append([nx2, ny2])
                l_shapes.append(current_l)

    classified_ls = []
    for l in l_shapes:
        if len(l) == 3:

            sorted_l = sorted(l, key = lambda point: (point[0], point[1]))
            x1, y1 = sorted_l[0]
            x2, y2 = sorted_l[1]
            x3, y3 = sorted_l[2]
            if x1==x2 and y2 == y3:
                classified_ls.append((sorted_l, 0)) #0 degree L
            elif y1 == y2 and x2 == x3:
                classified_ls.append((sorted_l, 90)) #90 degree L
            elif x1==x2 and y1==y3:
                classified_ls.append((sorted_l, 180))
            elif y1 == y2 and x1 == x3:
                classified_ls.append((sorted_l, 270))
        elif len(l) > 3:
            print("complex object found - skipping")

    return classified_ls

def add_blue_pixels(grid, l_shapes):
    new_grid = grid.copy()
    for l, orientation in l_shapes:
        sorted_l = sorted(l, key = lambda point: (point[0], point[1]))
        if orientation == 0:
            x1,y1 = sorted_l[0]
            x3,y3 = sorted_l[2]
            new_grid[x1, y1+1] = 1
            new_grid[x3,y3-1] = 1
        elif orientation == 90:
            x1,y1 = sorted_l[0]
            x3,y3 = sorted_l[2]
            new_grid[x1-1, y1] = 1
            new_grid[x3+1,y3] = 1
        elif orientation == 180:
            x1,y1 = sorted_l[0]
            x3,y3 = sorted_l[2]
            new_grid[x1, y1-1] = 1
            new_grid[x3, y3+1] = 1
        elif orientation == 270:
            x1,y1 = sorted_l[0]
            x3,y3 = sorted_l[2]
            new_grid[x1+1, y1] = 1
            new_grid[x3-1,y3] = 1
        else:
            print (f"unsupported orientation: {orientation}")
    return new_grid

After further debugging and analysis, I’ve identified several key issues in the previous versions of the find_l_shapes function:

  1. Duplicate Neighbor Checking: The code wasn’t correctly preventing duplicate neighbors from being added to the current L-shape, especially when multiple azure pixels were adjacent. This resulted in some pixels being processed multiple times and caused incorrect L-shape detection.

  2. Visited Set Management: I realized I was incorrectly managing the visited set, and duplicate additions were happening. I’ve added a check before adding a pixel to an L shape to correct this.

  3. Connected Component Logic: The algorithm’s core logic for determining whether a set of pixels formed a valid L-shape was flawed. I was checking for combinations that didn’t reliably identify L-shapes of different orientations.

I’ve rewritten the find_l_shapes function to address these issues. The updated code uses a more robust approach, first finding connected components of azure pixels and then checking if these components are valid L-shapes.

import numpy as np

def find_l_shapes(grid):
    def get_neighbors(x, y):
        neighbors = []
        for dx in [-1, 0, 1]:
            for dy in [-1, 0, 1]:
                if (dx, dy) == (0, 0):
                    continue
                nx, ny = x + dx, y + dy
                if 0 <= nx < grid.shape[0] and 0 <= ny < grid.shape[1]:
                    neighbors.append((nx, ny))
        return neighbors

    def is_l_shape(component):
        if len(component) != 3:
            return False

        xs, ys = zip(*component)
        xs = sorted(xs)
        ys = sorted(ys)

        return (xs[0] == xs[1] and ys[1] == ys[2]) or \
               (ys[0] == ys[1] and xs[1] == xs[2])

    azure_pixels = np.argwhere(grid == 8)
    visited = set()
    l_shapes = []

    for pixel in azure_pixels:
        x, y = pixel
        if tuple(pixel) in visited:
            continue

        component = []
        queue = [tuple(pixel)]
        visited.add(tuple(pixel))

        while queue:
            cx, cy = queue.pop(0)
            component.append([cx,cy])

            for nx, ny in get_neighbors(cx, cy):

                if 0 <= nx < grid.shape[0] and 0 <= ny < grid.shape[1] and grid[nx, ny] == 8 and (nx, ny) not in visited:
                    queue.append((nx, ny))
                    visited.add((nx, ny))

        if is_l_shape(component):
            l_shapes.append(component)

    return l_shapes
