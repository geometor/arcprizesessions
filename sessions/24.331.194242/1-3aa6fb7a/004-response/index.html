After further debugging and analysis, I’ve identified several key issues in the previous versions of the find_l_shapes function:

  1. Duplicate Neighbor Checking: The code wasn’t correctly preventing duplicate neighbors from being added to the current L-shape, especially when multiple azure pixels were adjacent. This resulted in some pixels being processed multiple times and caused incorrect L-shape detection.

  2. Visited Set Management: I realized I was incorrectly managing the visited set, and duplicate additions were happening. I’ve added a check before adding a pixel to an L shape to correct this.

  3. Connected Component Logic: The algorithm’s core logic for determining whether a set of pixels formed a valid L-shape was flawed. I was checking for combinations that didn’t reliably identify L-shapes of different orientations.

I’ve rewritten the find_l_shapes function to address these issues. The updated code uses a more robust approach, first finding connected components of azure pixels and then checking if these components are valid L-shapes.

Here’s the improved code:

import numpy as np

def find_l_shapes(grid):
    def get_neighbors(x, y):
        neighbors = []
        for dx in [-1, 0, 1]:
            for dy in [-1, 0, 1]:
                if (dx, dy) == (0, 0):
                    continue
                nx, ny = x + dx, y + dy
                if 0 <= nx < grid.shape[0] and 0 <= ny < grid.shape[1]:
                    neighbors.append((nx, ny))
        return neighbors

    def is_l_shape(component):
        if len(component) != 3:
            return False

        xs, ys = zip(*component)
        xs = sorted(xs)
        ys = sorted(ys)

        return (xs[0] == xs[1] and ys[1] == ys[2]) or \
               (ys[0] == ys[1] and xs[1] == xs[2])

    azure_pixels = np.argwhere(grid == 8)
    visited = set()
    l_shapes = []

    for pixel in azure_pixels:
        x, y = pixel
        if tuple(pixel) in visited:
            continue

        component = []
        queue = [tuple(pixel)]
        visited.add(tuple(pixel))

        while queue:
            cx, cy = queue.pop(0)
            component.append([cx,cy])

            for nx, ny in get_neighbors(cx, cy):

                if 0 <= nx < grid.shape[0] and 0 <= ny < grid.shape[1] and grid[nx, ny] == 8 and (nx, ny) not in visited:
                    queue.append((nx, ny))
                    visited.add((nx, ny))

        if is_l_shape(component):
            l_shapes.append(component)

    return l_shapes
